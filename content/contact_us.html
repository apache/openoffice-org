Thank you for contacting Apache OpenOffice

Please read the following options carefully. Sending a question to the wrong place can lead to a delayed response, or even cause your message to be ignored. Remember, we have millions of users of Apache OpenOffice. We want to help you. Please help us help you by directing your message to the right place.

  • If you are a user and need help with Apache OpenOffice...

    The fastest way to get expert help with OpenOffice is to use our Community Support Forums. You can search for answers to problems without registering. If you do register, you can post new questions to get help, participate in discussions, and help other users. The forum also allows for notifications on any post by subscribing to a topic.

    Or, send a message to our general support mailing list. (For full benefits, please see subscription information on the project mailing list page.)

    Other support mechanisms, including FAQs, commercial support, etc., are described on our Support Page.

  • If you want to submit a bug report...

    Note: Bug reporting is not a general way to receive support for Apache OpenOffice problems. If you need help using OpenOffice, please use one of the support options described in the first section. But if you have found a bug that you want to report you should use our Bugzilla issue tracking system.

    To submit a bug report, use our bug reporting system, Bugzilla. The QA issue handling page has useful information on getting started with Bugzilla, including searchng for issues, applying for a Bugzilla account, and enterring new issues.

  • For problems with the Apache OpenOffice insfrastructure...

    Note: These addresses are not for questions about the use of Apache OpenOffice. They are only for issues with the infrastructure services. User support questions sent to these addresses may be ignored.

    • For problems with the Bugzilla system, please contact us via BugZilla Admin mailing list.
    • For problems with the Wiki, please contact us via Development mailing list.
    • For problems with the Translation Pootle service, please contact us via L10N mailing list.

    Note: Emails sent to these mailing lists are publicly archived. So do not send personal information that you would not want on the web for anyone to see.

  • For all other inquiries...

    Please consult our mailing lists page for the appropriate mailing lists.
