Privileges you need for OOo.org QA

Just as a brief overview, the QA team is primarily focused on reviewing all the bug reports ( we sometimes call them issue reports ) that all the enthusiastic users of OpenOffice.org software submit.

In order to change any of the fields that a bug report has ( for example, the version of OpenOffice.org that the bug is found in ) you will need extra privileges on top of your default set of privileges that all OpenOffice.org user accounts have.


Obtaining extra IZ privileges

To obtain these extra privileges is simple:
1. Join the qa@openoffice.apache.org mailing list and check out the latest team news and topics.

You can learn more about the mailing list at:

http://qa.openoffice.org/join/mailinglists.html

2. Check out some issues that other active team members have worked on.

3. Add some comments to issues to get a feel for using IZ and how the process works.

4. Send an email to the qa@openoffice.apache.org mailing list with the subject line

  Volunteer Privileges - <Your OpenOffice.org Userid here>

  On of the team leads will shortly grant you privileges in IZ so that you can edit all the fields in a bug report.
