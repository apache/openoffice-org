:: com :: sun :: star :: system ::

unpublished constants group MailClientFlags
Usage Restrictions
not published
Description
These constants are used to specify how the XMailClient should behave.
Constants
DEFAULTS Uses the default settings when sending a mail, e.g. launches the current configured system mail client.  
NO_USER_INTERFACE Does not show the current configured system mail client GUI, but sends the mail without any further user interaction.  
NO_LOGON_DIALOG No logon dialog should be displayed to prompt the user for logon information if necessary.  
Constants' Details
DEFAULTS
const long DEFAULTS = 0;
Description
Uses the default settings when sending a mail, e.g. launches the current configured system mail client.
NO_USER_INTERFACE
const long NO_USER_INTERFACE = 1;
Description
Does not show the current configured system mail client GUI, but sends the mail without any further user interaction.

If this flag is specified, a recipient address must have been specified for the given XMailMessage object given to the method XMailClient::sendMailMessage().
NO_LOGON_DIALOG
const long NO_LOGON_DIALOG = 2;
Description
No logon dialog should be displayed to prompt the user for logon information if necessary.

When this flag is specified and the user needs to logon in order to send a mail message via the method XMailClient::sendMailMessage(), an ::com::sun::star::uno::Exception will be thrown.
