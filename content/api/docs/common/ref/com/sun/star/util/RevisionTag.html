unpublished struct RevisionTag
Usage Restrictions
not published
Description
represents the information that describes a revision of something, f.e. a document
Elements' Summary
TimeStamp contains the time when the revision was created ( can be invalid )  
Author contains an identifier for the author that created the revision( can be empty )  
Comment contains a comment that the author has left for this revision ( can be empty )  
Identifier contains a unique identifier for the revision and must not be empty

Description
contains a unique identifier for the revision and must not be empty

This identifier can have any form. It can be something like "1.2.3" or "Version 1" etc. It depends on the revision control system how it names the revisions.
