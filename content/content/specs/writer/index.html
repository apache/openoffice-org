Assigning Styles

The Styles control of the Text Object Bar offers a set of common used styles -- like Headings 1 to 3 -- even if they are not currently used in the document. The Styles control can also be used to define new paragraph styles based on the current selection. The relation of the Stylist control and the "Styles" tool window -- which used to be the "Stylist" in OOo1.1) -- is more tight. This is done by adding command items to the Styles control to open the "Styles" tool window. The commands to remove styles are aligned between Styles control and the Format menu, respectively the text context menu.

Frank Loehmann

The usability of the Insert - Caption dialog in OpenOffice 2.0 Writer will be enhanced.

The new designed dialog preview and the new possibility to add separators from the category label/number to the caption text itself prevents the user from making mistakes and saves rework time. General Undo Enhancements in Writer

Some core functionality is needed to be undoable. Examples are changing styles, tables, fields. The representation of actions in the undo/redo stack needs to be verbose enough to allow an unambiguous mapping to the actions. Not only the kind of action is to be displayed but also information about the object. E.g., if the user deletes several words by backspace, he gets an undo action for each word. Without additional information about the deleted word it is difficult to see which entry in the undo stack belongs to which deleted word.

Consider Paragraph Spacing on Table Borders

OOo 2.0 now considers spacing and indent set for left / right and for above / below paragraphs when calculating the entire spacing to a table's border. Those two sorts of spacing (paragraph and table) are now being added instead of only considering the distance to border set in the table's properties. This new feature improves the Microsoft interoperability and is also set as default for new created Writer documents to improve the export to Microsoft Word format. But this new feature has some side effects for new created documents: Writer automatically assigns paragraph styles "Table heading" and "Table content" to a table's heading and content. Due to paragraph spacing, set in those styles, the spacing between the last paragraph at the end of a cell and the table border is now much bigger than in OO 1.1. Especially single line paragraph content will not be shown vertically centered within the table cell anymore. Therefore the automatically assigned paragraph styles are changed to Default, the spacing is changed and repeated headings feature is turned off.

Frank Loehmann

ODF1.1: Use of soft-page-break

The ODF 1.1 allows the export of soft page breaks. A soft page break is a page break that has been inserted by a page oriented processor like Writer at a position where the document itself does not include a page break (e.g. paragraph attribute page break before/after). The soft page break information is for the purposes of consistent page breaks and for proper conversion to digital talking book formats.
