Java et OpenOffice.org

 

Java n'est pas obligatoire pour faire fonctionner correctement les applications d'OpenOffice.org. Cependant, certaines de ses fonctionnalités (détaillées ci-dessous) vous seront alors inaccessibles.
Règles pour l'utilisation de Java dans OOo : http://tools.openoffice.org/policies/java_usage.html

Vous pouvez installer une version contenant déjà Java, pour cela choisissez votre système  d'exploitation et votre région :


OpenOffice.org nécessite la version 1.4.1 ou supérieure pour fonctionner, pour télécharger la dernière version de Sun rendez-vous ici :

http://www.java.com/fr/download/manual.jsp

vous pouvez également utiliser Kaffe:
http://www.kaffe.org/

Suivez les instructions de téléchargement et d'installation sur votre ordinateur. Si vous installez OOo par la suite, celui-ci reconnaîtra alors l'environnement présent sur votre ordinateur et vous proposera de l'utiliser.

Pour installer Java après avoir installé OOo
Dans la version 1.1.x
Si vous avez déjà installé OOo et que vous souhaitez maintenant faire reconnaître Java par le logiciel, rendez vous sous < répertoire d'installation>/program/jvmsetup. 

En lançant le jvmsetup, OOo détectera la présence de la version java installée et vous proposera de l'utiliser.

<>Attention : Pour des questions de politique sur les licences, certains distributeurs de versions Linux ont désactivé les fonctionnalités Java de OOo. Il ne vous sera donc pas possible d'utiliser un environnement Java avec ces versions (ex : Fedora).

Dans la version 2.0.x
Installé normalement la version de Java et lorsque terminé :
- lancez OpenOffice.org
- Dans le menu Outils > Options > OpenOffice.org, sélectionnez Java. Dans la fenêtre de paramétrage, cochez  Utiliser un environnement d'exécution Java et sélectionnez celui devant être utilisé dans la fenêtre au-dessous.

Fonctionnalités requérant Java

Dans la version 1.1.x  :

  • utilisation des drivers JDBC
  • utilisation des filtres xslt, flat xml et petits portables
  • utilisation des applets Java dans OOo
  • langages basés sur Java dans le scripting framework
  • l'autopilote d'état
  • toutes les fonctions d'accessibilité
  • le testtool et QaTestDev
Pour la version 2.0, s'ajoute :
  • Base
    • Création à partir de l'assistant
  • Writer
    • Assistants (lettre, fax, agenda, HTML)
    • Enregistrer sous > AportisDoc (Palm)
    • Enregistrer sous > DocBook
    • Enregistrer sous > Pocket Word
  • Calc
    • Enregistrer sous  > Pocket Excel
  • Général
    • OfficeBean
    • Macros Java Script
    • Macros Beanshell
    • Python
Ce qui ne marche pas actuellement : Dans Outils > Macros > Gérer les macros > [JavaScript | Beanshell | Python] affiche une boîte de dialogue spécifiant qu'aucun JRE  n'est installé, mais finalement, la boîte de dialogue s'affiche. Voirs l'issue # 46680.
