La licence

Pour résumer, on peut dire que vous êtes pratiquement libres de tout vis-à-vis de la licence utilisée par OpenOffice.

Vous pouvez bien sûr utiliser OpenOffice gratuitement, en environnement professionnel comme pour un usage personnel. Vous pouvez aussi modifier, enrichir, voire améliorer le code du programme si vous en avez les compétences.

Ce n'est pas incroyable et ce n'est pas un piège. C'est le résultat d'un long travail de plusieurs centaines de milliers de développeurs de par le monde. Ces derniers ont travaillé sur une idée simple : le code source d'un logiciel lorsqu'il n'est pas accessible, permet à son éditeur d'enfermer ses utilisateurs dans une technologie propriétaire. Au final, cela bride et verrouille ces mêmes utilisateurs - ainsi que leurs développeurs - dans une logique inverse à la performance et à la rentabilité.

Ce constat fait, les développeurs adeptes de cet état d'esprit (souvent parmi les meilleurs) ont décidé de prôner l'ouverture aux éditeurs, en leur présentant les multiples avantages de ce qu'on appelle maintenant l'Open Source.

Le code source ouvert et rendu public permet à tous les développeurs de vérifier et améliorer le logiciel. Ce qui permet aux éditeurs à leur tour de bénéficier de l'expertise de centaines de milliers d'autres professionnels, chose irréaliste dans le cadre traditionnel d'un code source propriétaire et fermé.

Depuis le début des années 2000, l'industrie informatique à jauger l'immense potentiel de ce mode de pensée. Parmi les principaux projets, citons le plus connu : Mozilla avec son fameux navigateur Firefox. Il y a bien d'autres exemples comme le serveur Apache qui est le logiciel le plus répandu pour faire fonctionner des sites web ou les distributions Linux.

Sun Microsystems, en plaçant sa suite bureautique StarOffice dans ce circuit d'ouverture a été un précurseur : c'était effectivement la première fois qu'un logiciel commercial de cette importance était proposé à la communauté.

Jusqu'à la version 3.3 OpenOffice était sous licence GPL puis LGPL mais depuis la reprise en 2011 par la Fondation Apache (ASF), c'est leur licence qui est apposée.
