Note di Rilascio per Apache OpenOffice 3.4.1 (build 1372282)

Note Generali

La versione 3.4.1 di Apache OpenOffice è una versione di manutenzione, rilasciata con l'obiettivo di porre rimedio a problemi critici e a migliorare la qualità complessiva dell'applicazione.

Le aree generali di miglioramento includono: il supporto di lingue ulteriori, la correzione di errori, miglioramenti nelle prestazioni e aumento della compatibilità con Windows 8. Ulteriori dettagli riguardo ciascuna di queste aree sono riportati più avanti.

Per una panoramica più esaustiva dei cambiamenti rispetto alle versioni 3.3.X di OpenOffice.org e precedenti, è possibile vedere le Note di Rilascio relative a Apache OpenOffice 3.4.0.

Supporto di lingue ulteriori

Le nuove traduzioni disponibili in Apache OpenOffice 3.4.1 sono le seguenti:

  • Finlandese
  • Inglese UK
  • Khmer
  • Slovacco
  • Sloveno

Le traduzioni aggiornate (rispetto alla versione 3.4.0) disponibili in Apache OpenOffice 3.4.1 sono le seguenti:

  • Olandese
  • Spagnolo
  • Italiano
  • Ungherese

Per un elenco completo delle lingue disponibili e dei language packs è possibile vedere: http://www.openoffice.org/download/other.html

Piattaforme Supportate

Le seguenti piattaforme continuano ad essere disponibili come versioni ufficiali di Apache OpenOffice: Microsoft Windows, MacOS (Intel), Linux (32 bit), Linux (64 bit).

Inoltre numerosi membri della comunità stanno lavorando a "port" per Solaris, FreeBSD e OS/2, e a versioni "portable" o "live". Per maggiori informazioni vedere: http://www.openoffice.org/porting/ .

Correzione di errori

Alla data del 16.08.2012 c'erano 69 problemi della versione 3.4.0 risolti e verificati.

La lista si trova qui: http://s.apache.org/Huv

È richiesto l'accesso a OpenOffice Bugzilla per visionarli.

(La creazione di un account su OpenOffice Bugzilla, possibile per chiunque, vi permetterà di ricercare qualsiasi segnalazione di errore che possa interessare).

Le correzioni più importanti sono state effettuate nei seguenti settori:

  • problemi di "Quickstart"
  • pulizia di estensioni
  • notifica automatica degli aggiornamenti ora nuovamente attiva per impostazione predefinita
  • operazioni di "Annulla" in Calc
  • operazioni di "Ripristina/Ripeti" in Calc
  • problemi di blocco di Calc e Writer se l'estensione LanguageTool è attiva
  • mancato riconoscimento di Java: è stato aggiunto "Oracle" come provider di Java per facilitare l'identificazione JVM (Java Virtual Machine). Il collegamento precedente era con "Sun".

Il team di controllo qualità ha seguito la storia di altre correzioni. Una panoramica completa, in inglese, è su:

http://wiki.services.openoffice.org/wiki/QA/Report/WeeklyReport

Inoltre, dalla versione 3.4.1, Apache OpenOffice usufruisce dell'analisi statica fornita da Coverity per scoprire eventuali difetti del programma prima che il codice sia rilasciato. Un grazie particolare a Coverity per aver autorizzato l'utilizzo di questa loro utility già vincitrice di premi!

Miglioramento di funzionalità e prestazioni

  • pulizia delle aree temporanee utilizzate dall'installazione di estensioni, riducendo così, nelle installazioni multiutente, l'area disco necessaria per ogni utente.
  • solo un dizionario di inglese per installazioni US-en al posto delle 5 nella versione 3.4.0 con riduzione di spazio occorrente sull'hard-disk. Gli utenti potranno aggiungere altri dizionari di inglese (canadese, australiano, neozelandese) se desiderano.
  • aggiornamento della libreria "Hunspell" alla versione 1.3.2 dalla 1.2.9. La nuova versione contiene un'assistenza migliorata, specie per le lingue danese, tedesco e altre di ceppo germanico, nell'utilizzo di parole composte.

Compatibilità con Microsoft Windows 8

Con questa versione abbiamo fatto un test con Windows 8. Finora sembra che vada bene. Siamo in procinto di iniziare la check list di Certificazione ufficiale di Windows 8 e rimangono ancora alcuni problemi minori. I dettagli si trovano sul wiki:

http://wiki.services.openoffice.org/wiki/Documentation/Windows_App_Certification_Kit_-_Test_Results_for_Apache_OpenOffice_3.4

Nota: i miglioramenti fatti in Apache OpenOffice 3.4.1 per supportare Windows 8 andranno a beneficio anche degli utenti di Windows 7.

Problemi noti

  • Apache OpenOffice 3.4.0 e 3.4.1 gestiscono il profilo utente in modo diverso dalle versioni precedenti. Il vecchio profilo utente è automaticamente convertito in modo che gli utenti possano mantenere le proprie estensioni e impostazioni. In un ridotto numero di casi, specialmente con profili altamente personalizzati (con molte estensioni o personalizzazioni) la conversione non riesce. I sintomi più comuni sono: frequenti crash dell'applicazione, problemi con i dizionari o i sinonimi, apertura e crash di OpenOffice dopo pochi secondi. Per risolvere questi problemi, basta ripristinare/rinominare il profilo utente come spiegato sul forum ufficiale di OpenOffice: http://user.services.openoffice.org/it/forum/viewtopic.php?f=6&t=187
  • Apache OpenOffice 3.4.0 e 3.4.1 saranno segnalati dal nuovo Gatewatcher in OS X Mountain Lion. Si tratta di una nuova funzionalità che contribuisce alla protezione da malware nei sistemi Mac. Presso il link indicato di seguito è indicata una procedura per far funzionare le applicazioni non installate dal Mac app store. Leggere l'articolo su http://support.apple.com/kb/HT5290.
  • Apache OpenOffice 3.4.0 e 3.4.1 supportano Java 7, che è la configurazione consigliata; ma (specialmente su Windows a 64-bit) si potrebbero ricevere notifiche che avvertono che la versione di Java è difettosa. In questo caso, basta scaricare e installare jre-6u32-windows-i586.exe e riconfigurare OpenOffice tramite Strumenti- Opzioni- OpenOffice.org - Java. Per ulteriori informazioni, vedere http://user.services.openoffice.org/en/forum/viewtopic.php?f=15&t=54974.
  • Il pacchetto di installazione di Apache OpenOffice 3.4.1 per Windows estrarrà i file di installazione e lancerà immediatamente il wizard di installazione. In rare circostanze, probabilmente a causa delle incompatibilità con altri programmi installati, il wizard di installazione potrebbe non partire automaticamente. In questo caso, aprire la cartella "OpenOffice Installation Files" sul Desktop e fare doppio click sul file "setup.exe" per iniziare la procedura di installazione.
