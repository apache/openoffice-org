Guys,

There seem to be reoccurring discussions regarding OOos Java usage, e.g. see

http://www.fsf.org/news/open-office-java.html [and] //servlets/ReadMsg?listName=dev&msgNo=13179

I am really convinced, that we did everything to enable anybodies Java (even none at all), by

a) defining OOos Java usage policy (see http://tools.openoffice.org/policies/java_usage.html )

b) by implementing the Java Vendor framework (see http://udk.openoffice.org/common/man/spec/javavendorextension.sxw )

c) by providing help when needed in respect to some legacy code, which partly makes use of some undocumented sun classes

and d) by certainly ensuring that OOo even runs without Java properly (e.g. Java can be deselected during the build process, leaving out Java based features, e.g. applets)

I would appreciate, if you can provide the latest state of the GCJ builds and if there are any obstacles left, which can not easily be fixed by some lines of code. We are certainly willing, and it will be a pleasure, to help and to work together with you to sort these issues out.

Please follow up on jdk at tools.openoffice.org

Thanks in advance

Kay
