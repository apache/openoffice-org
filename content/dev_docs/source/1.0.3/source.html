General Information

  • Note: The download mirrors listed here are volunteer. If there are errors in the URL, please send a note to dev@website.openoffice.org.
  • Download Central This page includes links to documents related to building and using OpenOffice.org.
  • OpenOffice.org 1.1 Our advanced releases includes features and elements that you will like.
  • OpenOffice.org SDK
  • System Requirements  Our system requirements don"t change from build to build; these were written for the 632 build. Please read these before downloading.
  • Features  In addition to our Releases Notes (see link below), which change for each build, we have also compiled a Features page, which concisely lists the more important features of the OpenOffice.org application.
  • Release Notes Please review the Release Notes to learn what is new and what has changed about this build.
  • Md5Sums Please check your downloads against the md5sums. For information on md5sums, we have an instruction and resource page.
  • Project Introduction A general and useful page introducing the project and how you can start to make a difference.

Information on Downloading and Building the Source

  • On Downloading Source. The source files above are snapshots from CVS. We recommend that before you start a build you do a CVS update to get any changes made since the snapshot was taken.
  • Solver: Output Tree. If you do not wish to do a full build, you can download the solver [output tree] and checkout the individual CVS modules you are interested in. The individual CVS modules can be built against the output tree. After a successful build, the CVS module binaries and/or shared libraries are delivered to the output tree. See the build instructions for details.
  • Testing the Build. In order to test your work, you will need to download and install the binaries, and copy your new binaries and/or shared libraries into the OpenOffice.org installation.
  • Checking Out Source from CVS. We have written an explanation on how to check out the full source code from the CVS tree for a given tag. For the adventurous only, you can check out the latest source by omitting the tag in the instructions on this page. There"s no guarantee it will build, however.
  • Browsing Source Code  The top navbar of each project has a link, "Source code," that takes you to a Web-viewable layout of CVS modules particular to that project.
