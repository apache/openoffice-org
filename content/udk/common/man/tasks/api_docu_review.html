UDK IDL API

Motivation

The documentation of all exported API functions, classes and interfaces must be reviewed for the final API. We split it in a two pass process. The first pass should be done by the developer of the API function|class|interface and the second pass should be done by a native speaker to get a well formed documentation. For this we provide this list of all exported header|IDL files where the ongoing of this process can be documented. Each person who made a review of the documentation should mark this for the appropriate header|IDL file.
