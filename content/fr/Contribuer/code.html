Contribuer au code d'OpenOffice.org

Toutes les contributions apportées au projet OpenOffice.org pour aider les développeurs à parfaire le logiciel, doivent être soumises via IssueZilla.

Rendez-vous sur le wiki des développeurs pour obtenir les toutes dernières informations sur la meilleure façon de commencer. 

À titre d'information complémentaiver, vous pouvez aussi lire le guide du hacker d'OpenOffice.org et également les livres blancs afin d'avoir un aperçu technique d'OpenOffice.org.

Enfin visitez les projets pour connaître ceux auxquels vous souhaitez contribuer et consultez les indications et conseils pour une meilleur compréhension de la structure des projets

Voici comment proposer votre code source pour le logiciel au projet OpenOffice.org

Les codes sources doivent être proposés comme "PATCH" et seront envoyés au responsable du projet approprié via IssueZilla

  1. Remplissez le formulaire de copyright, SCA
    il en existe une version  PDF .

  2. Pour que le code soit accepté :
    1. Pour un code de 10 lignes ou plus, votre formulaire de copyright doit avoir été reçu.
      Vous pouvez consulter la liste des tenants de copyright approuvés ici.

    2. Le responsable du projet doit avoir approuvé votre proposition de code.

    3. Votre code doit être sous licence LGPLv3 ou supérieure.
Si vous ne souhaitez pas travailler sur les sources et que votre développement reste externe (exemple add-on en StarBasic), vous devez le placer sous code LGPL pour qu'il soit mis à disposition sur le site OpenOffice.org. Vous pouvez sinon le soumettre sous forme d'extensions sur le site Extensions d'OpenOffice.org, la licence est alors à votre choix.

Nous espérons que les francophones feront encore parler de la francophonie dans le projet OpenOffice.org !
Bien sûr nous vous accompagnons à chaque étape de cette intégration. N'hésitez pas à prendre contact avec nous sur la liste dev<at>fr.openoffice.org
