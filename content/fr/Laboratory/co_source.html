Nécessités pour le téléchargement

Le système que vous téléchargez doit posséder :
  • une connexion internet avec un vitesse minimu de 56 ko/sec
  • 400 Mo d'espace disque pour le code source seul
  • Un des systèmes suivants :
    • FTP
    • CVS 1.10 ou supérieur

Télécharger les sources

Vous devez télécharger le code source avant de pouvoir commencer à compiler. Pour télécharger les sources, voir la section Télécharger les sources de la section téléchargement central

Vous pouvez soit télécharger les sources complètes, soit télécharger seulement le solver (output tree). Le solver (output tree) vous permet de compiler des modules CVS individuels, plutôt que la suite complète. Pour plus d'information, voir Aperçu du process de compilation.

Télécharger les sources complètes

Nous estimons que cela va prendre environ 8 heures pour télécharger l'intégralité des sources pour une connexion avec un modem de 56 ko.
Vous pouvez télécharger un tarball de l'intégralité du code source à partir de la section Télécharger les sources de la section téléchargement central .
Alternativement, vous pouvez utiliser CVS pour télécharger la dernière version du code source. Pour faire un check out des sources en utilisant CVS, vous devez installer la version CVS 1.10 ou supérieure.
Les sections suivantes décrivent comment faire un check out du code source pour chaque sytème d'exploitation :
  • Linux
  • Solaris
  • Win 32
  • Mac OS X
Check out du Code Source pour Linux
Pour faire un check out du dernier code source pour Linux, suivez la procédure suivante
  1. Paramètrez la variable d'environement $CVSROOT de la façon suivante :
     
    % setenv CVSROOT :pserver:anoncvs@anoncvs.services.openoffice.org:/cvs
  2. Loguez-vous sur CVS et entrez anoncvs comme mot de passe
  3. Pour faire un checkout du code, entrez une commande telle que celle-ci
    % cvs co -r milestone-build OpenOffice
    remplacez miles-stone build par le numéro de la version que vous souhaitez télécharger. L'exemple suivant vous montre les commandes pour télécharger la versions 609
    % setenv CVSROOT :pserver:anoncvs@anoncvs.services.openoffice.org:/cvs

    % cvs login

    (Logging in to anoncvs@anoncvs.services.openoffice.org)

    CVS password: anoncvs

    % cvs co -r OpenOffice609 OpenOffice

    Note : Vous pouvez faire un check out de la dernière version en ommettant le tag, c'est à dire en exécutant la commande suivante :
    %     cvs co OpenOffice

Check out du Code Source pour Win 32
Pour faire un check out de la dernière version du code source pour un système Win 32, suivez les étapes suivantes :
  1. Paramètrez la variable d'environnement pour le CVSROOT de la façon suivante :
    C:\> set CVSROOT=:pserver:anoncvs@anoncvs.services.openoffice.org:/cvs
  2. Loguez vous à CVS en entrant anoncvs  comme mot de passe
  3. Paramètrez la variable d'environnement HOME vers un répertoire approprié. Cette variable est paramètrée par défaut sous Linux et Solaris, mais vous devez la définir manuellement sour les systèmes d'exploitation Win 32,. Si vous ne le faites pas, cela peut poser des problèmes avec CVS.
  4. Pour faire un check out du code, entrez une commande similaire à celle-ci :
    C:\> cvs co -r milestone-build OpenOffice
    Remplacez milestone-build par le numéro de la version que vous voulez télécharger. Par exemple, pour télécharger la version 609, entrez les commandes suivantes :
    C:\> set CVSROOT=:pserver:anoncvs@anoncvs.services.openoffice.org:/cvs

    C:\> cvs login

    (Logging in to anoncvs@anoncvs.services.openoffice.org)

    CVS password: anoncvs

    C:\> set HOME=C:\temp

    C:\> cvs co -r OpenOffice609 OpenOffice

    Note : vous pouvez faire un check out de la version en ommettant le tag et en entrant la commande suivante

    C:\> cvs co OpenOffice

Télécharger le Output Tree (solver)
Pour télécharger la dernière version du code source, suivez ces étapes
  1. Téléchargez le tarball de la version solver à partir du site OpenOffice.org. Ce tarball est nommé solverxxx_oscpu.tar.gz , où xxx signifie le numéro de la version,  os signifie le système d'exploitation et cpu signifie l'architecture hardware. Par exemple, la version du tarball pour la version 69 pour Linux ou Intel est appellée :
    solver609_linuxintel.tar.gz.
  2. Faites un check out du projet sur lequel vous souhaitez travailler. Vous devez faire un check out de la version du projet qui correspond à la même version. Par exemple, si vous faites un checkout de la version solver609_linuxintel.tar.gz  , alors vous devez faire un check out du projet portant le numéro de version 609.
    Pour faire un check out du module, entrez une commande similaire à la commande suivante :
    % cvs co -r milestone-build project-name
    remplacez le nom project-name par le nom du projet  que vous souhaitez télécharger. Par exemple pour faire un check out du module CVS Visual Class Libraries, entrez les commandes suivantes :
    3. % cvs co -r OpenOffice609 vcl
