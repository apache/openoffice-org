Java & Apache OpenOffice, OpenOffice.org

完整的OpenOffice(OpenOffice.org)的功能需要方可实现. Java主要被使用的功能是HSQLDB数据库引擎(被Apache OpenOffice Base使用)，可访问性和辅助技术.此外一些向导也依赖于Java技术.查看额外的话题关于Java在OpenOffice.org和Apache OpenOffice中的使用请参见安装FAQ

所以，这对我来说意味着什么？ Base（数据库组件）完全依赖Java技术来运行，但其他程序（如Writer，Calc和Impress）只需要Java来实现特殊功能。 如果你不使用Base并不想使用向导，那么你就不需要安装Java来运行Apache OpenOffice（和旧版本的OpenOffice.org）。 您可以完全阻止OpenOffice提示是否使用Java，让OpenOffice不使用Java运行时环境（JRE）需要从OpenOffice的主菜单中进行以下操作： 工具 -> 选项 -> OpenOffice.org -> Java, 反选"使用Java运行时环境".但是，我们建议在您的系统上最好有一个Java运行时环境以便充分利用OpenOffice的功能。

旧版本的OpenOffice.org（3.3以下）包括一个JRE打包的下载。最新版的Apache OpenOffice 3.4.1没有。

如果你已经在标准的Java安装位置装有JRE, OpenOffice应该能检测到并可以在 工具->选项->OpenOffice.org -> Java 可以看到和启用. 如果没有被检测到，您也可以手动在这个菜单里添加.

如果你在没有JRE(或者只是忘了安装)的情况下配置的OpenOffice，你可以在这之后安装Java，丢失的功能依然可以使用.

当前版本，Apache OpenOffice 3.4,可以使用 Oracle Java和openJDK可靠工作在众多环境下.从这些网站下载JRE如果你觉得你需要这些功能。

目前的版本，Apache OpenOffice 3.4，在64位Windows 7系统中使用32位JRE已有报告的问题。但是，这个问题不能在所有情况下被复制。从Oracle下载的Java1.7是最新版本的Java。

请通过我们的错误报告系统报告关于在OpenOffice中使用Java您所遇到的任何问题， Bugzilla,或者在 支持 页中列出的其它支持场所,无论是在用户列表或论坛.
