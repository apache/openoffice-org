2. starting implementations

FS suggested that we should start implementations of the new common Office UI as soon as the factory design is finished, and not wait until all people are done with their tasks for the upcoming drop.

CL urged to consider all accessibility requirements which have to be fullfilled by the team's work (and not in the lower VCL layers). The main demur is that without this, we may be forced to do changes twice: once in the old code, and once in the new code which is to replace the old one. One way to prevent this would be to really force all applications to move to the new code base as soon as it is available, without a transition time.
