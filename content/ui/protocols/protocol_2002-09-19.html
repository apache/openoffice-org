2 Roundtable

2.1 Jump to Text Cursor Position

CJ and FL are analyzing the current behavior when and how SO/OO.o scrolls the view to the cursor position. Sometimes this causes an annoying effect, e.g. on saving a document the window scrolls back to the text cursor.

2.2 Save Cursor Position in Writer Document

Some user like that SO/OO.o Writer opens documents at the position, where the document was last saved. Some other users dislike this feature. Esp. on distributing SO/OO.o documents to other users this is an unintended quality.

Action Item for CJ: Summarize the pros and cons and propose an UI spec for this topic.

  • Read-only documents should always open in the default position

  • Cursor position can/should be stored in the history. This makes the state relative to each SO/OO.o user.

2.3 Last Document Close Behavior

MMP is working on a new definition for the behavior "What should happen if the user closed the last document in SO/OO.o?" Proposal will be posted to OO.o' Web site, soon.
