2 Round table

2.1 Testing HC readiness

The automated testing will provide tools for doing screenshots from all known dialogs, one for HC and one for “normal” mode. This will ease checking the coverage of our HC implementations a lot. The test is then to be run and done by CJ.

2.2 The help viewer lacks some functionality for Accessibility. Current ideas are:

  • 3 separate options

    • Formatting On/Off

    • Colors On/Off

    • Font Sizing On/Off

  • alternatively: let the user provide an own style sheet to be used when displaying help content



This will probably lead to an own options page dedicated to the help content, where the new options are to be merged with the already existent onces, which reside on other pages currently.

CL is to file a bug for this.

2.3 Splitters

There was a discussion about the Accessibility of Splitter windows. There exists a proposal to move splitters by using Alt + Cursor keys, but this has the disadvantage of conflicting with traveling within CTL-glyphs (which uses Alt-Cursor-Left/-Right, too). No sufficient solution has been found, yet.
