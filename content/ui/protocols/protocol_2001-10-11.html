2. Replaceable context menus

OS decided that replaceable context menus are importent for Final6.0. An interface design has not been decided yet. We argued about using an XSelectionSupplier instead of actually creating the current selection each time for context menus. This solution has some pros if the listener for context menus is not always interested in the selection. We also agreed that we should use an container interface to transport menus instead of xml or property sequence because the latter would force us to transport the complete menu for every listener all the time, even if the listener is not interested in the actuall menu content.

OS pointed out that CD will do the implementation for the interfaces. Who designs the interfaces is not yet decided.

3. New source versions to check in new features

FS found out that there are no decisions to have a new source version for post final 6.0 features. According to FS, MA said that there is no real need for this until we have a concept for our ui project structure. OS pointed out that this has not a very high priority as long as we have so many final bugs.

4. Movement of UI code from sfx

FS informed us that he had a talk with mba about the posibility to move ui code from the sfx project. This should not be a real problem in general but there are some problems with user settings which have high dependencies with the view implementations in sfx. MBA said that this can be solved with factories.
