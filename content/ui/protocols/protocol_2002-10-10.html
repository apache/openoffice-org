1.1 Reorganize document section in ui.openoffice.org

The documents section in http://ui.openoffice.org/servlets/ProjectDocumentList offers some inconveniences that have to be overcome by moving the proposals and specifications to an area where the files can be checked in using the cvs system.

The problems are:

  • Each document needs an approval from the project owner. During the time the new or changed document is not approved no access to the document is possible.

  • Prior versions of documents cannot be retrieved. That makes it harder to find differences and almost impossible to find out which version of a specification has been valid for a given milestone of the application.

  • Often SourceCast (the used system for OOo web site) has destroyed documents in the update process.

2 Roundtable

FS mentioned problems in mouse selection of edit fields as a result of CTL and BiDi changes made at those fields. He will file a task regarding the problem.

MMP brought up a discussion about the documentation area in ui.openoffice.org.

The necessity of documents to be accepted by the project owner leads to problems in the specification of features. When the project owner is not reachable the document cannot be published. Prior versions of a specification are also not accessible between the upload and the acceptance of the new version.

To overcome this the proposals and specifications will be moved so that they can be checked in using cvs access.
